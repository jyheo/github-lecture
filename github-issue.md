## Github Issue
* 누구나 issue를 만들 수 있음
	- 기능 제안, 버그 리포팅 등등
* 프로젝트 collaborator들이 issue의 중요도에 따라 label을 붙일 수 있음
	- 버그, 도움, 기능 추가 등
* 개발자가 issue를 직접 해결하고 close하거나 다른 사람에게 assign할 수 있음
* 어떻게든 해결하고 나면 close
